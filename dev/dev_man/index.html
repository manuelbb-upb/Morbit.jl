Morbit.CompositeVecFun — Type
CompositeVecFun(; inner_ref, outer_ref,
    inner_nl_index = nothing, outer_nl_index = nothing,
    num_outputs = num_outputs( outer_ref ) 
)

A CompositeVecFun is a special AbstractVecFun to use for functions $f:ℝⁿ→ℝᵏ, f = φ ∘ g$, where $g:ℝⁿ → ℝᵐ$ is an expensive function (that is possibly used elsewhere too) and $φ:ℝᵐ→ℝᵏ$ is an outer function. Hence, when evaluating a CompositeVecFun, the value of the function referenced by inner_ref is plugged into outer_ref. When evaluating the overall jacobian at $x$, the chain rule provides

\[\begin{aligned} ℝ^{k×n} \ni Df(x) &= Dφ(g(x))Dg(x), \quad Dφ ∈ ℝ^{k×m}, Dg ∈ ℝ^{m×n}, ∇f_ℓ(x) &= (Dφ(g(x))Dg(x))_{ℓ,:}^T = (∇φ_ℓ(g(x))^T Dg(x))^T = Dg(x)^T ∇φ_ℓ(g(x)).\]

For the Hessian of output $f_ℓ$ it holds that

\[Hf_ℓ(x) = D(∇f_ℓ(x)) = D(Dφ_ℓ(g(x)) \cdot Dg(x)) = (Hφ_ℓ(g(x)) \ Dg(x))\cdot Dg(x) + Dφ_ℓ(g(x)) \cdot Hg(x)\]

Morbit.LagrangeConfig — Type
LagrangeConfig(; kwargs... )

Configuration for Lagrange Polyoniaml models.

  • degree

    Degree of the surrogate model polynomials. Default: 2

  • θ_enlarge

    Enlargement parameter to consider more points for inclusion. Default: 2.0

  • LAMBDA

    Quality parameter in Λ-Poisedness Algorithm. Default: 1.5

  • allow_not_linear

    Whether or not the interpolation sets must be Λ-poised (and the models fully linear). Default: false

  • optimized_sampling

    Whether or not to try to construct a new interpolation set in each iteration. Default: true

  • save_path

    Path to look at for a pre-saved poised set of interpolation points in [0,1]^n. Default:

  • io_lock

    Lock to use parallel setups if multiple models can access save_path. Default: nothing

  • algo1_max_evals

    Maximum number of polynomial evaluations to find a poised set. Default: -1

  • algo2_max_evals

    Maximum number of polynomial evaluations to make a set Λ-poised. Default: -1

  • algo1_solver

    NLopt Solver to use for the poisedness problem. Default: :LN_BOBYQA

  • algo2_solver

    NLopt Solver to use for the Λ-poisedness problem. Default: :LN_BOBYQA

  • max_evals

    Maximum number of evaluations allowed to the true objective(s). Default: typemax(Int64)

Morbit.RbfConfig — Type
RbfConfig(; kwarg1 = val1, … )

Configuration type for local RBF surrogate models.

  • kernel

    RBF kernel (Symbol), either :cubic, :inv_multiquadric, :multiquadric, :exp or :thin_plate_spline. Default: :cubic

  • shape_parameter

    RBF shape paremeter, either a number or a string containing Δ. Default: NaN

  • polynomial_degree

    Degree of polynomial attached to RBF. -1 means no polynomial. Default: 1

  • θ_enlarge_1

    Local enlargment factor of trust region for sampling. Default: 2

  • θ_enlarge_2

    Maximum enlargment factor of maximum trust region for sampling. Default: 2

  • θ_pivot

    Sampling parameter to generate Λ-poised set. The higher, the more poised. Default: 1 / (2θenlarge1)

  • θ_pivot_cholesky

    Parameter for 2nd sampling algorithm to ensure boundedness of Cholesky factors. Default: 1.0e-7

  • require_linear

    Require models to be fully linear in each iteration. Default: true

  • max_model_points

    Maximum number of training sites. -1 is reset to 2n+1. Default: -1

  • use_max_points

    Sample new sites to always use the maximum number of points. Default: false

  • optimized_sampling

    Whether or not to re-construct the training set in each iteration. Default: true

  • max_evals

    Maximum number of objective evaluations. Default: typemax(Int64)

Morbit.RefSurrogate — Type
RefSurrogate(model_ref, output_indices)

A RefSurrogate holds a reference model_ref to some AbstractSurrogate object and delegates most of the evaluation to this inner model. RefSurrogates are meant to model a single (vector-valued) objective or constraint function. As an AbstractSurrogate might model more than one objetive or constraint, we have a field output_indices for filtering out the right output indices from the inner model output.

Morbit.TaylorConfig — Type
TaylorConfig(; degree, gradients :: RFD.CFDStamp, hessians :: RFD.CFDStamp, max_evals)

Configuration for a polynomial Taylor model using finite difference approximations of the derivatives. By default we have degree = 2 and gradients == hessians == RFD.CFDStamp(1,2), that is, a first order central difference scheme of accuracy order 3 is recursed to compute the Hessians and the gradients. In this case, the finite difference scheme is the same for both Hessians and gradients and we profit from caching intermediate results.

Morbit.get_poised_set — Method
get_poised_set( basis, points; solver = :LN_BOBYQA, max_solver_evals = -1 )

Compute a point set suited for polynomial interpolation.

Input:

  • basis: A vector of polynomials constituting a basis for the polynomial space.
  • points: (optional) A set of candidate points to be tried for inclusion into the poised set.
  • solver: NLopt solver to use. Should be derivative-free.
  • max_solver_evals: Maximum number of evaluations in each optimization run.

Return:

  • poised_points :: Vector{T} where T is either a Vector{F} or an SVector{n_vars, F} and F is the precision of the points in points, but at least Float32.
  • lagrange_basis :: Vector{<:AbstractPolynomialLike}: The Lagrange basis corresponding to poised_points.
  • point_indices: An array indicating which points from points are also in poised_points. A positive entry corresponds to the index of a poised point in points. If a poised point is new, then the entry is -1.
